# Tutor Report Generator — Project Instructions

Веб-сервис для автоматической генерации резюме тьюторских встреч. Тьютор вставляет транскрипцию → Claude API анализирует → готовый отчёт для вип-менеджера.

## Архитектура

**Один файл.** Весь проект — `index.html`. HTML, CSS и JavaScript в одном файле. Никакого сборщика, никакого бэкенда, никаких npm-зависимостей. Открывается напрямую в браузере через `file://`.

Единственная внешняя зависимость — mammoth.js (CDN): парсит `.docx` транскрипции в текст.

## Структура файлов

```
tutor-report-generator/
├── index.html          # Весь проект
├── raw-data/           # Автосохранённые транскрипции (.txt), gitignored
├── .claude/
│   ├── SKILL.md        # AI-скилл: логика анализа транскрипций (системный промпт)
│   └── settings.local.json
├── .gitignore          # Исключает raw-data/ и .DS_Store
└── README.md
```

## Как запустить

```bash
open index.html
# или
open -a "Google Chrome" index.html
```

Никакого `npm start`, сервера или порта. Просто открыть файл в браузере.

При первом открытии: нажать **⚙ Настройки** → вставить Anthropic API Key (`sk-ant-...`). Ключ сохраняется в `localStorage` под именем `trg_api_key`.

## Ключевые функции в index.html

| Функция | Строка | Что делает |
|---|---|---|
| `generate()` | ~630 | Основная — вызывает Claude API, рендерит результат |
| `renderResult(text)` | ~687 | Парсит markdown-ответ в HTML |
| `mdToHtml(text)` | ~694 | Легковесный md→HTML конвертер |
| `saveTranscript(text)` | — | Сохраняет транскрипцию в `raw-data/` |
| `updateCounter()` | ~537 | Считывает и показывает счётчик встреч из localStorage |
| `incrementCounter()` | ~551 | +1 к счётчику после успешной генерации |
| `processFile(file)` | ~581 | Обрабатывает .txt и .docx файлы |
| `launchConfetti()` | — | Конфетти при зелёном студенте |

## Системный промпт и модель

- **Системный промпт**: `const SYSTEM_PROMPT` (~строка 393) — подтягивается из `SKILL.md` (встроен в код).
- **Модель**: `claude-sonnet-4-6`
- **API endpoint**: `https://api.anthropic.com/v1/messages`
- Заголовок `anthropic-dangerous-direct-browser-access: true` обязателен для браузерных запросов.

Если нужно изменить логику анализа — редактируй `SYSTEM_PROMPT` в `index.html`. `SKILL.md` в `.claude/` — это документация, не код.

## localStorage

| Ключ | Что хранит |
|---|---|
| `trg_api_key` | Anthropic API key |
| `trg_count` | Счётчик сгенерированных отчётов |

## Как вносить изменения

**Стиль**: CSS-переменные в `:root` (строки 10-24). Цвета, радиусы — всё там.

**Логика генерации**: только в функции `generate()` и `SYSTEM_PROMPT`.

**Анимации**: `launchConfetti()` для 🟢, red flag логика — внутри `renderResult()`.

**Правило**: не добавлять внешних зависимостей без крайней необходимости. Проект должен открываться без интернета (кроме CDN mammoth.js и вызова API).

## Как проверять ошибки

Для статического HTML нет линтера — проверяй вручную:

1. **Открыть в браузере** → `open index.html`
2. **DevTools Console** (F12) — нет ли ошибок при загрузке
3. **Проверить API-запрос**: вставить короткий текст, нажать «Сгенерировать», посмотреть Network → XHR
4. **Типичные ошибки**:
   - `401` — неверный или просроченный API key
   - `CORS` — только если открывать через `http://`, не `file://`
   - `mammoth is not defined` — проблема с CDN, нет интернета

## Публикация (git workflow)

Репозиторий: `https://github.com/flashkate/tutor-report-generator`

```bash
git add index.html README.md   # не добавлять raw-data/
git commit -m "короткое описание изменения"
git push
```

Перед пушем убедиться:
- `raw-data/` не попала в staged (там реальные транскрипции студентов — приватные данные)
- API key не захардкожен в коде (должен быть только в localStorage)
- `index.html` открывается без ошибок в браузере

## Команды

- `/run` — открыть сервис в браузере
- `/check` — проверить файл перед публикацией
- `/publish` — подготовить коммит и запушить

## Контекст проекта

Пользователь — тьютор онлайн-университета Zerocoder, ведёт вводные встречи с VIP-студентами. 25 встреч в месяц. Студенты — программы AI-консалтинг, AI-студия, AI-стартап.

Резюме публикуется в закрытый Telegram-чат для вип-менеджеров.
