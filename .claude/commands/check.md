Проверь проект перед публикацией. Выполни последовательно:

**1. Структура файлов**
```bash
ls -la
```
Убедись: есть `index.html`, `README.md`, `.gitignore`. Папка `raw-data/` присутствует локально, но должна быть в `.gitignore`.

**2. Безопасность — API key не захардкожен**
```bash
grep -n "sk-ant-" index.html
```
Результат должен быть пустым. Если нашлось — немедленно убери ключ из кода.

**3. raw-data не попадёт в коммит**
```bash
git status
```
В списке staged/unstaged не должно быть файлов из `raw-data/`. Если есть — они содержат реальные транскрипции студентов (приватные данные).

**4. Синтаксис .gitignore**
```bash
cat .gitignore
```
Должны присутствовать строки `raw-data/` и `.DS_Store`.

**5. Актуальность README**
Прочитай `README.md` и сравни с текущим функционалом `index.html`. Если есть расхождения — обнови README до коммита.

По итогам проверки: сообщи что нашлось и что нужно исправить перед пушем.
